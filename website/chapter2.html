Installing and Configuring Tembo

Getting started with Tembo is designed to be straightforward, but proper configuration is crucial for maximizing your team's productivity. This section will walk you through the installation process and essential configuration steps that will set the foundation for successful AI-assisted development.

Prerequisites

Before installing Tembo, ensure your development environment meets these requirements:

  • Node.js 18 or higher installed on your system
  • Git configured with your credentials
  • Access to your project repository with appropriate permissions
  • A supported IDE or text editor (VS Code, IntelliJ, or similar)

Installation Process

Tembo can be installed through multiple methods depending on your team's preferences and infrastructure setup:

Method 1: npm Installation

For most JavaScript/TypeScript projects, the npm installation method provides the fastest setup:

npm install -g @tembo/cli
tembo init

Method 2: Docker Setup

For teams preferring containerized environments or working with multiple language ecosystems:

docker pull tembo/ai-engineer:latest
docker run -v $(pwd):/workspace tembo/ai-engineer init

Understanding Tembo's Capabilities and Limitations

To work effectively with Tembo, it's essential to understand both what it excels at and where human oversight remains critical.

Core Strengths

  • Pattern Recognition: Tembo excels at identifying and implementing common programming patterns
  • Code Consistency: Maintains consistent coding style across your entire project
  • Comprehensive Testing: Generates thorough test suites including edge cases
  • Documentation Generation: Creates detailed documentation that stays in sync with code changes
  • Refactoring Safety: Performs large-scale refactoring while preserving functionality

Areas Requiring Human Guidance

  • Business Logic Validation: Complex domain-specific rules need human verification
  • Architecture Decisions: High-level system design choices benefit from human experience
  • User Experience: Interface design and user flow optimization require human insight
  • Performance Trade-offs: Complex performance optimization decisions need human evaluation

Setting Up Your Development Environment

Proper environment configuration is crucial for seamless AI-assisted development. This section covers the essential setup steps for maximum productivity.

IDE Integration

Tembo integrates with popular development environments to provide real-time assistance:

VS Code Extension

The VS Code extension provides the most comprehensive Tembo experience:

  • Real-time code suggestions and completions
  • Integrated chat interface for feature requests
  • Automatic code review and improvement suggestions
  • Seamless integration with existing workflows

Command Line Interface

For teams preferring terminal-based workflows, the CLI provides full functionality:

tembo generate component UserProfile
tembo review --file src/components/UserProfile.tsx
tembo test --coverage --watch

Team Configuration

Setting up Tembo for team collaboration requires coordination around coding standards, review processes, and integration workflows:

Shared Configuration

Create a .tembo.json file in your project root to ensure consistent behavior across team members:

{
  "codeStyle": "typescript-strict",
  "testFramework": "jest",
  "reviewLevel": "thorough",
  "integrations": {
    "linear": true,
    "sentry": true,
    "github": true
  }
}

Code Review Integration

Configure Tembo to participate in your existing code review process:

  • Automatic pull request creation with detailed descriptions
  • AI-powered code review comments and suggestions
  • Integration with GitHub, GitLab, or Bitbucket workflows
  • Customizable review criteria and quality gates
