Chapter 2: Getting Started with Tembo

Create Your First AI-Assisted Feature in GitHub

The best way to understand Tembo's power is by experiencing the complete value loop of AI-assisted software development: conceive → define → create → accept. This chapter will guide you through creating your first feature using Tembo's GitHub integration.

Step 1: Choose or Create a GitHub Repository

To get started with Tembo, you'll need a GitHub repository to work with. We recommend creating a new repository specifically for this tutorial to keep things clean and focused.

Creating a New Repository

Follow these steps to create a new GitHub repository:

  1. Navigate to github.com/new
  2. Choose any name you like for your repository (e.g., "my-first-tembo-project", "job-board-features", etc.)
  3. Important: Check the "Add a README file" option to initialize your repository
  4. Leave the repository public for easier setup (you can change this later)
  5. Click "Create repository"

Your new repository will be created with a basic README.md file, providing the perfect starting point for your first AI-assisted feature.

Step 2: Sign Up for Tembo and Connect Your Repository

Now that you have a GitHub repository, it's time to connect it to Tembo and unlock AI-powered development capabilities.

Tembo Account Setup

  1. Visit tembo.io and click "Sign Up"
  2. Sign up using your GitHub account for seamless integration
  3. Complete the onboarding process
  4. When prompted, connect your newly created repository to Tembo
  5. Grant the necessary permissions for Tembo to create branches and pull requests

Once connected, Tembo will have access to your repository and can begin assisting with your development tasks.

Step 3: Your First AI-Assisted Feature

Now for the exciting part—let's use Tembo to create a meaningful feature that demonstrates the full AI-assisted development workflow.

Example Task: Job Board Feature List

We'll create a comprehensive feature list for a job board website, organized by priority. This is a perfect example of how AI can help with product planning and feature definition.

Creating the Task in Tembo

  1. Log into your Tembo dashboard
  2. Navigate to your connected repository
  3. Click "Create Task" to open the task creation window
  4. Copy and paste this prompt into the task window:

Task Prompt:
"Generate a comprehensive feature list for a job board website, and add this list to the repository. Prioritize the features into 3 priority buckets: High Priority (must-have core features), Medium Priority (important but not critical), and Low Priority (nice-to-have enhancements). Include features for both job seekers and employers. Format the output as a well-structured markdown file that can serve as a product roadmap."

Understanding the AI Process

After submitting your task, Tembo will:

  • Conceive: Analyze your requirements and understand the scope
  • Define: Break down the task into specific, actionable components
  • Create: Generate the feature list and create a new file in your repository
  • Present: Submit the work as a pull request for your review

Step 4: Review the Pull Request

Once Tembo completes your task, you'll receive a pull request containing the generated feature list. This is where the collaborative aspect of AI-assisted development shines.

Reviewing AI-Generated Code

Navigate to your repository's "Pull Requests" tab to see Tembo's work. You'll find:

  • A detailed pull request description explaining what was created
  • The new markdown file with your job board feature list
  • Features organized into the three priority buckets you requested

Requesting Changes

If you want to modify or expand the feature list, you can:

  • Add comments directly to the pull request
  • Tag @tembo in your comments to request specific changes
  • Ask for additional features, different prioritization, or formatting changes

For example, you might comment: "@tembo Please add mobile app features to the feature list and include user authentication requirements for each priority bucket."

Step 5: Merge the Pull Request

Once you're satisfied with the feature list, it's time to complete the development cycle by merging the pull request.

The Merge Process

  1. Review the final changes in the pull request
  2. Click "Merge pull request" to accept the AI-generated work
  3. Choose your preferred merge method (merge commit, squash and merge, or rebase and merge)
  4. Add a merge commit message if desired
  5. Click "Confirm merge"

Completing the Value Loop

Congratulations! You've just completed your first full cycle of AI-assisted software development:

  • ✅ Conceive: You identified the need for a job board feature list
  • ✅ Define: You provided clear requirements and prioritization structure
  • ✅ Create: Tembo generated comprehensive, well-organized content
  • ✅ Accept: You reviewed, potentially refined, and merged the results

What You've Accomplished

Through this hands-on exercise, you've experienced the core benefits of AI-assisted development:

  • Speed: Generated comprehensive features in minutes, not hours
  • Completeness: AI considered aspects you might have initially overlooked
  • Structure: Received well-organized, professional documentation
  • Collaboration: Maintained control through the review and approval process
  • Integration: Seamlessly worked within your existing GitHub workflow

This foundational experience prepares you for more complex AI-assisted development tasks. In the next chapter, we'll explore advanced techniques for collaborating with AI on actual code implementation.
